# Harmony PNG DEM 协议（QGIS 导出规范）

这份文档定义了 Harmony 规划地形导入时，唯一受支持的 PNG 高度图导出协议。

## 目的

Harmony 可以直接导入 GeoTIFF DEM 文件；但当源数据必须导出为 PNG 高度图时，PNG 必须遵循一套固定协议，这样编辑器才能稳定解析正海拔和负海拔，而不需要对每个文件单独手工调参。

这套协议是严格的。如果导出的 PNG 不符合下面的规则，Harmony 会在导入时直接拒绝。

## 固定海拔映射

所有 PNG DEM 导出都必须使用同一套映射：

- 灰度值 `32` = `0m`
- 每 `1` 个灰度级 = `20m`
- 可表示的最低海拔 = `-640m`
- 可表示的最高海拔 = `4460m`

Harmony 对每个地形像素使用下面的公式解码：

```text
elevationMeters = (grayValue - 32) * 20
```

示例：

- 灰度 `0` -> `-640m`
- 灰度 `32` -> `0m`
- 灰度 `57` -> `500m`
- 灰度 `255` -> `4460m`

## 文件要求

导出的文件必须同时满足以下条件：

- 格式：`PNG`
- 像素类型：`8-bit grayscale`
- Alpha：全图完全不透明
- 颜色：只能是灰度，不允许 RGB 偏色

整张 PNG 都必须是实际地形数据，不需要额外的校验行、边框或协议标记。

## 导出前的范围检查

在把 DEM 转成 PNG 之前，先确认源栅格的海拔范围落在 `-640m` 到 `4460m` 之间。

如果任意像素超出这个范围：

- 不要 clamp
- 不要重新缩放映射
- 不要修改协议
- 先停下来处理源数据

如果悄悄 clamp 或重新缩放，导入后的地形一定会错。

## QGIS 工作流

下面的步骤默认源 DEM 的单位是米。

### 1. 载入源 DEM

- 在 QGIS 中打开 DEM。
- 确认 DEM 的海拔单位是米。
- 检查栅格统计信息，确认 min/max 都落在 `-640m` 到 `4460m` 之间。

### 2. 把海拔转换为协议灰度值

打开 Raster Calculator，用下面的表达式把海拔转换为 byte 灰度：

```text
round(("DEM@1" / 20) + 32)
```

注意事项：

- 项目里的 DEM 波段名可能不是 `DEM@1`，请替换成实际的波段引用。
- 计算结果必须保持在整数范围 `0` 到 `255`。
- 如果输入数据超出了支持的海拔窗口，应该先修正源数据，而不是修改公式。

### 3. 将转换后的栅格导出为 PNG

- 把 Raster Calculator 的结果导出为 PNG。
- 输出必须是灰度图。
- 确保导出的 PNG 完全不透明。
- 不要应用色带、阴影、对比度拉伸或透明度。

### 4. 保持整张图都是地形数据

不要额外添加校验行、边框或其他协议标记。导出的整张 PNG 都必须是实际地形数据。

## 交付检查清单

在导入 Harmony 之前，请逐项确认：

- 文件是 `.png`
- PNG 是 `8-bit grayscale`
- PNG 完全不透明
- 整张图全部都是地形数据
- 地形数据仍然遵循 `gray 32 = 0m` 和 `20m per gray`

## Harmony 的导入行为

当 Harmony 导入一个合法的 PNG DEM 时：

- 剩余像素会按固定公式解码
- PNG 的 min/max 海拔会被视为固定协议元数据
- 编辑器不再允许手工编辑 PNG 的海拔范围

## 常见失败原因

如果出现以下任一情况，Harmony 会拒绝该 PNG：

- 文件不是 PNG
- 图片不是灰度图
- 任意像素带有透明度
- 最终地形栅格小于 `2x2`

## 备注

- 这套协议只适用于 PNG 高度图。
- 当需要更高精度或原生地理参考时，优先使用 GeoTIFF 导入。
- 不要为不同项目随意发明新的映射，Harmony 只支持这里定义的固定协议。
